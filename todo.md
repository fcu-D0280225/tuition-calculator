## 待辦清單

### 高優先（家長／學生／老師端的核心體驗）

- [ ] **家長／學生端上課打卡**：學生用 QR code 或登入後一鍵打卡，自動寫入 `lesson_records.status`；老師端只需事後核對
  - 影響：要新增「家長／學生」帳號類型 + 公開／半公開的打卡頁
- [ ] **調課／補課專屬 UI**：在某筆 lesson 上「改期」，自動標註原本日期 + 新日期，與請假整合（請假後可從佇列裡挑日子補回）
  - 影響：可能要在 `lesson_records` 加 `original_date` / `parent_lesson_id` 欄位
- [ ] **親師溝通**：在「學生」頁面下加備忘錄／訊息序列（`student_messages` 表：student_id / author_user_id / body / created_at）
- [ ] **成績紀錄**：`student_scores` 表（學生、科目、分數、考試名稱、日期、學校／補習班來源、是否公開展示）+ 學生頁面下 tab

### 中優先（行政效率）

- [ ] **線上繳費（第三方支付）**：先做 LINE Pay / 街口介接 PoC（手續費高但快）
  - 後續視量再評估銀行 API 直連
  - 繳費成功 webhook 自動寫 `payment_records`
- [ ] **遲繳自動通知**：cron 每天檢查未繳帳單超過 N 天，產生通知（先 console / email；之後再串 LINE Notify）
- [ ] **員工薪資核算**：新增 `employees` 表（非老師人員）+ 月薪 / 時薪 / 出勤紀錄 + 薪資結算
- [ ] **招生情況分析**：新增 `inquiries` 表（問班紀錄：來源、日期、家長、學生、有興趣課程、是否成單、未成單原因）+ 漏斗分析報表
- [ ] **學校行事曆連動**：先支援手動上傳 ICS（學校／教育局發的）；自動匯入下一階段

### 低優先（加值體驗）

- [ ] **課後作業紀錄**：先做純紀錄（標題／指派日期／截止日期／是否完成），線上練習之後再說
- [ ] **訂餐功能**：和繳費 / 帳單整合（`meal_orders` 表）
- [ ] **AttendancePage 行動體驗**：手指 hit area、離線暫存（IndexedDB）、上線同步
- [ ] **損益報表 PDF / 匯出**：可下載當月 P&L
- [ ] **學費收費單**：加 LINE 分享文字模板（直接貼訊息）
- [ ] **教材成本對家長**：學費收費單加教材費明細區塊
- [ ] **雜支自動化**：固定支出（房租）排程每月自動產生
- [ ] **預算／目標**：每月預算對比實績儀表板
- [ ] **多分校 / 多教室**：未來擴大時加 `branch_id` 維度
- [ ] **稽核紀錄**：誰改了哪筆、何時改的（audit log）

## 待釐清的設計問題（先想清楚再寫 code）

- **家長 / 學生帳號要做嗎？** 目前只有「老師 / 一般使用者 / 管理員」。家長/學生端打卡、看作業、看成績、繳費都需要這個帳號類型；如果不做，要靠 share token + 家長家用 LINE 通知。
- **第三方支付選哪家？** LINE Pay 介接快但手續費 ~3%，銀行 API 便宜但審核時間長。先做 PoC 還是直接申請？
- **學校行事曆連動**：要支援哪幾所學校？手動上傳 ICS 還是定期爬？
- **成績紀錄是否對家長／學生開放？** 公開的可拿來招生（用學生匿名化版），但隱私要小心。
